Todos los formularios en web, manejan para el envío de las diferentes peticiones que realiza el usuario. Debemos recordar que la mayoría de las aplicaciones en web trabajan sobre una arquitectura cliente servidor. Significa que el cliente realiza una petición (request) y el servidor debe atender mediante una respuesta (response). Los métodos más comúnes son: GET:

Este método se encarga de enviar la información del formulario por medio de la salida o url estándar del navegador, esto propicia que pueda haber inyecciones de código realizando violaciones a la seguridad del sistema.

POST:

Este método se encarga de enviar información igual por el método estándar: pero mediante un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo del sistema.

GET solo se usa para operaciones de lectura. POST sirve para cambiar la información de un recurso por parte del servidor. (Insertar un registro es lo más común). DELETE sirve para eliminar un recurso en específico. PUT almacena la información de un recurso particular para su actualización. HEAD regresa el código de respuesta y cualquier cabecera HTTP asociada.
